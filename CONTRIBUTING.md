# Contributing to MilkTea

First off, thank you for considering contributing to **MilkTea**! We appreciate your interest in improving this project and making it better for everyone.

## How to Contribute

There are several ways you can contribute to this project:

### 1. Reporting Bugs

If you find a bug, please check the existing issues to see if it's already been reported. If not, create a new issue with the following information:

- **Steps to reproduce** the bug.
- **Expected behavior** vs **actual behavior**.
- **Screenshots** (if applicable).
- **Environment details** (OS, browser, version, etc.).

### 2. Suggesting Enhancements

We welcome suggestions for new features or improvements. Please create a new issue with a detailed description of the enhancement you would like to propose.

### 3. Submitting Code Changes

We follow the steps below for submitting code changes:

1. **Fork the repository**: Click the "Fork" button at the top right of the project page.
2. **Clone your fork**: Use `git clone <your_fork_url>` to clone the repository to your local machine.
3. **Create a new branch**: Always create a new branch for each feature or bug fix. For example:
   ```bash
   git checkout -b feature/awesome-feature
   ```
4. **Make your changes**: Implement your changes, and ensure that you’ve tested them locally.
5. **Commit your changes**: Write a clear, concise commit message following the **Conventional Commits** format. For example:
   ```bash
   git commit -m "feat: add awesome feature"
   ```
6. **Test your changes**: Ensure that you’ve written unit tests for your changes and that they pass before submitting your PR.
7. **Push to your fork**: Push your changes to your forked repository.
8. **Create a Pull Request**: Open a pull request from your fork to the main repository. Be sure to describe the changes you made and reference the relevant issue number (if any).

### 4. Code Style

We follow certain coding conventions in this project to maintain consistency:

- Use **[Prettier](https://prettier.io/)** for code formatting.
- Follow the **[Conventional Commits](https://www.conventionalcommits.org/)** standard for commit messages.
- Write **unit tests** for any new features or bug fixes.
- Ensure all tests pass before submitting your pull request.
- Keep changes minimal and focused. We appreciate clean, concise commits!

### 5. Code of Conduct

By participating in this project, you agree to abide by the project's [Code of Conduct](CODE_OF_CONDUCT.md). Please be respectful and considerate in all interactions.

## Thank You!

We appreciate your help in making **MilkTea** better! Every contribution, big or small, is welcome and valued. Thank you for your time and effort in helping improve this project!

If you have any questions, feel free to open an issue or reach out to the project maintainers.

**Good first issues**: If you’re new to contributing, check out our [Good First Issues](#) for easy tasks to get started with!
